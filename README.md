# 《To Change 蛻化》— Foundry VTT 遊戲系統

把無骰、以塔羅大阿爾克那為核心的「蛻化」主題 TRPG《To Change》搬上 Foundry VTT 的遊戲系統。

- **系統 id**：`to-change`
- **相容版本**：Foundry VTT **v11 – v13**（minimum 11、verified 13）
- **語言**：繁體中文（主）、English

> 本系統依《To Change》（Ulysses · Ewen 著）規則打造,規則文字取自同目錄下的非公式繁體中文翻譯專案。
> 卡圖與十字／輪盤插圖取自原書 PDF,**打包供你自己的牌桌使用**;若要公開散佈本系統,請先向原作者取得美術與內容授權。

---

## 功能

| 功能 | 說明 |
|---|---|
| **蛻化角色卡** | 四屬性（MF／MA／PF／PA）、可切換「蛻化十字」與「跨性別蛻化輪盤」兩種版面 |
| **行動占卜引擎** | 完整自動化:依未受損點數算抽牌數,處理援助／昇位／墮位／高塔／正義加減牌,翻牌選牌,自動套用 22 張預兆牌效果(死神、節制、惡魔、高塔、月亮、太陽、審判、世界等特例),蛻變,結果發聊天卡 |
| **預兆牌牌組** | 首次啟動自動建立 22 張附卡圖的 Cards 牌組,供桌面瀏覽查閱 |
| **單人遊玩模式** | 起始牌、每日事件、依累積蛻化量分歧的正／逆位事件、蛻化日誌 |
| **隨機表格擲表器** | 表 1–5（隨機角色×2、隨機蛻化×2、小事件),依「洗牌取底牌」方式擲表 |
| **故事動作設定** | 14 條故事動作可在世界設定中勾選,顯示於角色卡作為提示 |

## 安裝

本系統未上架官方套件庫,可用下列任一方式安裝。

### 方式 A:資訊清單網址自動安裝(建議)

1. 開啟 Foundry VTT 設定畫面 →「**遊戲系統(Game Systems)**」分頁 →「**安裝系統(Install System)**」。
2. 在下方「**資訊清單網址(Manifest URL)**」欄位貼上:

   ```
   https://raw.githubusercontent.com/ericwang01129/To-Change-fvtt-system/main/system.json
   ```

3. 按「安裝」,Foundry 會自動下載並解壓到 `Data/systems/to-change/`。
4. 建立新世界時選擇「To Change 蛻化」即可。

### 方式 B:手動安裝

1. 找到 Foundry 的使用者資料夾 `Data/`(設定畫面 → Configuration 可看到路徑)。
2. 把整個系統資料夾複製或 symlink 到 `Data/systems/`,並**改名為 `to-change`**:

   ```
   <FoundryUserData>/Data/systems/to-change/
   ```

3. 重新啟動 Foundry → **Game Systems** 分頁應出現「To Change 蛻化」。
4. 建立新世界時選擇此系統即可。

> 首次進入世界時,GM 端會自動建立「預兆牌」牌組與 5 張隨機表格(只執行一次)。

## 使用方式

### 建立角色

- 建立 Actor（型別 `character`）→ 開啟角色卡。
- 在標頭選擇**角色卡版面**:蛻化十字 或 跨性別蛻化輪盤。
- 在「主要」分頁挑選**面相／昇位／墮位**三張角色牌(點卡面選牌)。

### 記錄傷害與蛻化

- **蛻化十字**:左鍵點格 = 蛻化(由外而內),右鍵點格 = 傷害;中央永恆之點有專屬切換鈕。
- **跨性別蛻化輪盤**:點任一格切換該跨性別蛻化面向;永恆為手動切換。
- 屬性面板的紅色圓點也可直接點按調整傷害。

### 行動占卜

1. 在屬性面板按該屬性的「**占卜**」鈕。
2. 確認受測屬性、填入援助人數,按「占卜」抽牌。
3. 翻開的牌會亮出;點一張作為命運,或按「**蛻變**」以一點蛻化換成功。
4. 系統自動結算、處理牌堆、把結果發到聊天。

### 單人遊玩

- 角色卡「單人遊玩」分頁勾選啟用 → 開啟單人面板。
- 選**起始牌** → 按「進行新的一天」:指定當天蛻化屬性、抽事件牌、寫日誌。

### 隨機表格 / 重置牌堆

- 左側場景控制列(v11–v12)有「隨機表格」與「重置預兆牌」按鈕。
- v13 或找不到按鈕時,可用 macro 呼叫:`game.toChange.openTableRoller()`、`game.toChange.Deck.reset()`。

## 安裝後煙霧測試清單

1. 系統載入無主控台錯誤;Game Systems 出現本系統。
2. 建立角色 → 切換十字／輪盤 → 點格記錄受損與蛻化,數值有存檔。
3. 對某屬性占卜 → 抽牌數 = 未受損點數,可選牌、可蛻變,聊天卡正確輸出。
4. 驗證特例牌:死神(僅唯一牌可選)、節制(棄牌洗回)、惡魔(成功+蛻化)、高塔(下次+2)、月亮(抽底牌)、世界(重抽)。
5. 啟用單人模式 → 選起始牌 → 進行一天 → 事件牌輸出到聊天。
6. 開隨機表格擲表器 → 各表擲一次、可發到聊天。

## 已知限制

- 22 張**面相牌特殊能力**僅自動化機制單純者(愚者重洗、正義對稱抽牌、高塔棄全部翻開牌);其餘以提示文字呈現,由玩家／敘事者手動套用。
- **故事動作**僅作為提示顯示,不改寫占卜引擎行為。
- 占卜以「單次進行一場」為前提;牌堆狀態由 GM 端統一寫入,玩家發起的占卜經 socket 中繼,需有在線 GM。
- 採 classic Application API 以相容 v11;在 v13 會出現 deprecation 警告但功能正常。
- 牌組以世界 Cards 文件交付;未提供跨版本 compendium pack。

  system.json / template.json / to-change.mjs
  module/        引擎與應用程式(deck, resolution, solo, 角色卡 sheet …)

  styles/        to-change.css
